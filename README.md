# Тренажёр устного счёта (Math Trainer)

Десктопное приложение для тренировки математических навыков: генерирует примеры по выбранному типу и уровню сложности, проверяет ответы, считает время и точность, ведёт статистику сессии. Аналог Human Benchmark, но для математики.

## Что умеет приложение

- **4 типа заданий** (каждый — отдельный генератор):
  - Арифметика (`+ − × ÷`);
  - Обыкновенные дроби (сложение/вычитание, ответ сокращается);
  - Проценты (найти `P%` от числа);
  - Линейные уравнения вида `a·x + b = c`.
- **3 уровня сложности**: лёгкий / средний / сложный (меняются диапазоны
  чисел и набор операций).
- **2 режима**: обычная тренировка и режим «на время» с обратным отсчётом.
- **Статистика сессии**: количество верных/неверных ответов, точность в %,
  среднее и общее время.
- **Настройки сохраняются** между запусками в JSON-файл.
- Полноценный **CLI** у консольной версии и **горячие клавиши + F1-справка**
  в GUI.

## Архитектура проекта

Главный принцип — **вся логика отделена от интерфейса** и вынесена в
библиотеку классов `MathTrainer.Core`. Библиотека не ссылается ни на один
GUI-фреймворк, поэтому её можно переиспользовать в **Console, WinForms, WPF,
MAUI, Avalonia**.

```
            ┌─────────────────────────────┐
            │      MathTrainer.Core        │   ← библиотека классов (вся логика)
            │  генераторы, сессия, модели, │
            │   настройки, статистика      │
            └──────────────┬──────────────┘
                           │ ссылаются
          ┌────────────────┴────────────────┐
          │                                  │
┌─────────▼──────────┐            ┌──────────▼─────────┐
│ MathTrainer.Console │            │  MathTrainer.Wpf   │
│  CLI + интерактив   │            │  графический UI    │
└─────────────────────┘            └────────────────────┘
```

Один и тот же класс `TrainingSession` управляет тренировкой и в консоли, и в
окне WPF. Интерфейсы общаются с ним **через события** (`ProblemPresented`,
`AnswerChecked`, `SessionFinished`) — поэтому ядро ничего не знает о
конкретном UI.

## Объектная модель (ООП)

Проект демонстрирует основные средства C#:

| Средство языка        | Где используется                                                            |
|-----------------------|-----------------------------------------------------------------------------|
| **Классы**            | `Problem`, `TrainingSession`, `AppSettings`, генераторы и т.д.               |
| **Свойства**          | `Problem.Text`, `Session.Score`, `AppSettings.Difficulty` …                  |
| **События**           | `ProblemPresented`, `AnswerChecked`, `SessionFinished` в `TrainingSession`   |
| **Коллекции**         | `List<AnswerRecord>` истории ответов, `IReadOnlyList<>` в статистике         |
| **Наследование**      | `ArithmeticGenerator : ProblemGeneratorBase` (и ещё 3 наследника)           |
| **Абстрактный класс** | `ProblemGeneratorBase` — общая инфраструктура генераторов                    |
| **Интерфейсы**        | `IProblemGenerator` — единый контракт для всех генераторов                   |
| **Полиморфизм**       | `TrainingSession` работает с любым `IProblemGenerator` через `Next()`        |
| **Перечисления**      | `ProblemType`, `Difficulty`, `SessionMode`                                   |
| **Фабрика**           | `GeneratorFactory.Create(...)` скрывает создание конкретных классов          |

## Как запустить проект

> Требуется **.NET SDK 10.0** или новее. Проверить: `dotnet --version`.

Сначала соберите всё решение из корневой папки проекта:

```powershell
dotnet build MathTrainer.slnx -c Debug
```

### 1. Запуск графического интерфейса (WPF)

**Вариант А — из Visual Studio 2026:**
1. Откройте `MathTrainer.slnx`.
2. В обозревателе решений щёлкните правой кнопкой по проекту
   **MathTrainer.Wpf → Set as Startup Project**.
3. Нажмите **F5** (запуск с отладкой) или **Ctrl+F5** (без отладки).

**Вариант Б — из терминала PowerShell:**
```powershell
dotnet run --project src/MathTrainer.Wpf
```

Откроется окно: выберите тип заданий, сложность и режим, нажмите **«Старт»**
(или **Enter**) и решайте примеры. Справка — клавиша **F1**.

> Примечание: WPF работает только на Windows. На вашей системе (Windows 11)
> всё запускается без дополнительных настроек.

### 2. Запуск консольной версии и параметры командной строки

**Интерактивный запуск с параметрами по умолчанию:**
```powershell
dotnet run --project src/MathTrainer.Console
```

**Запуск с параметрами командной строки:**
```powershell
# Дроби, средний уровень, 15 заданий
dotnet run --project src/MathTrainer.Console -- --type Fractions --level Medium --count 15

# Проценты, сложный уровень, режим «на время» с лимитом 90 секунд
dotnet run --project src/MathTrainer.Console -- -t Percentages -l Hard -m Timed --time 90

# Только сохранить выбранные настройки (без запуска тренировки)
dotnet run --project src/MathTrainer.Console -- --type Equations --save-only

# Справка по всем параметрам (аналог F1)
dotnet run --project src/MathTrainer.Console -- --help
```

Можно также запустить уже собранный `.exe` напрямую:
```powershell
.\src\MathTrainer.Console\bin\Debug\net10.0\MathTrainer.Console.exe --type Arithmetic --count 5
```

**Полный список параметров:**

| Параметр            | Короткий | Значения                                           | По умолч.   |
|---------------------|----------|----------------------------------------------------|-------------|
| `--type`            | `-t`     | `Arithmetic`, `Fractions`, `Percentages`, `Equations` | `Arithmetic` |
| `--level`           | `-l`     | `Easy`, `Medium`, `Hard`                           | `Easy`      |
| `--count`           | `-c`     | положительное число                                | `10`        |
| `--mode`            | `-m`     | `Practice`, `Timed`                                | `Practice`  |
| `--time`            |          | секунды (для `Timed`)                              | `60`        |
| `--save-only`       |          | только сохранить настройки и выйти                 | —           |
| `--help`            | `-h`     | показать справку                                   | —           |

Во время сессии: введите ответ и нажмите **Enter**; `q` — досрочный выход.

## Горячие клавиши

В графическом интерфейсе:

| Клавиша | Действие                                   |
|---------|--------------------------------------------|
| **Enter** | Начать сессию / подтвердить ответ        |
| **Esc**   | Остановить текущую сессию                 |
| **F1**    | Показать справку по управлению и CLI      |

## Настройки пользователя

Выбранные параметры (тип, сложность, режим, количество заданий, лимит времени)
автоматически сохраняются и подхватываются при следующем запуске.

Файл настроек: `%AppData%\MathTrainer\settings.json`
(обычно `C:\Users\<имя>\AppData\Roaming\MathTrainer\settings.json`).

Формат — читаемый JSON:
```json
{
  "ProblemType": "Fractions",
  "Difficulty": "Medium",
  "Mode": "Practice",
  "QuestionCount": 15,
  "TimeLimitSeconds": 60
}
```

Консольная и графическая версии используют **один и тот же** файл настроек.

## Сценарии для демонстрации и проверки

Так как итоговое тестирование выполняется вручную, ниже — готовые сценарии:

1. **Базовая тренировка (GUI):** запустить WPF → тип «Арифметика», сложность
   «Лёгкий», 5 заданий → ответить на все → проверить итоговое окно со
   статистикой.
2. **Проверка неверного ответа:** ввести заведомо неправильный ответ →
   убедиться, что показывается правильный (красным/оранжевым).
3. **Режим «на время» (GUI):** режим «На время», лимит 15 секунд → не
   отвечать → убедиться, что по истечении времени сессия завершается.
4. **Дроби с сокращением:** тип «Дроби» → проверить, что ответы вроде `2/4`
   принимаются как `1/2`.
5. **Сохранение настроек:** выбрать тип «Уравнения» и режим «На время»,
   закрыть и снова открыть приложение → параметры сохранились.
6. **CLI:** `... -- --type Percentages --level Hard --count 3` → пройти 3
   задания в консоли → увидеть итоги.
7. **Справка:** в GUI нажать **F1**; в консоли запустить `--help`.
